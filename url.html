URLs (Uniform Resource Locators)

What it is

A URL is the address of a resource on the internet. Every web page, image, video, or file accessible online has a URL. It tells your browser exactly where to go and how to get there.

Anatomy of a URL

A URL has several parts:

  • Protocol: The method used to access the resource (e.g., http, https)
  • Domain: The server where the resource is located (e.g., www.example.com)
  • Port: The specific port number on the server (e.g., :8080)
  • Path: The location of the resource on the server (e.g., /path/to/resource)
  • Query String: Additional parameters for the request (e.g., ?key=value)
  • Fragment: A section within the resource (e.g., #section)

URLs vs. URIs

You will sometimes hear the term URI (Uniform Resource Identifier). All URLs are URIs, but not all URIs are URLs. A URI identifies a resource; a URL identifies it and provides a way to locate it.
